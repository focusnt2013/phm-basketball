# 智智慧投篮机系统 - 完整开发计划

## 一、项目概述

### 1.1 产品定位
基于传统投篮机改造的智能投篮训练系统，通过Android一体机实现人脸无感登录、手势隔空交互、数字化计分和运动榜单展示。

### 1.2 核心功能
- 用户身份识别（人脸离线比对）
- 手势交互控制（模式切换、启动确认）
- 进球自动检测（NPN光电传感器信号采集）
- 实时计分统计（出手数、命中数、命中率）
- 数据云端同步（本地缓存+断网补传）
- 运动榜单展示（WebView网页加载）

### 1.3 硬件组成

| 设备 | 型号/规格 | 功能说明 |
|------|-----------|----------|
| 光电传感器 | 世宇KL-GY40JMFS1 | 红外反射式，进球检测 |
| 信号转换模块 | USB转TTL+光耦隔离 | 电平转换与信号采集 |
| Android一体机 | 算力盒子 | 主控系统，触控显示 |

### 3.2 用户身份模块

**功能描述**
- 接收后台推送的用户人脸特征值
- 本地人脸库管理（增删改查）
- 摄像头实时检测，自动识别登录用户
- 支持老师/学生不同身份角色
- 支持游客模式（未注册用户）

**数据模型**

```kotlin
data class User(
    id: String,              // 用户ID
    name: String,            // 姓名
    role: String,            // 角色：teacher/student/visitor
    grade: String?,          // 年级（学生）
    className: String?,      // 班级（学生）
    faceFeature: FloatArray, // 人脸特征值
    createdAt: Long,        // 创建时间
    updatedAt: Long         // 更新时间

sealed class UserState {
    object NoUser : UserState()                    // 场景榜单状态
    object Detecting : UserState()                 // 人脸检测中

    object Visitor : UserState()                   // 游客模式
```

**技术方案**
- 人脸特征值存储：Room Blob字段
- 比对算法：百度智能云离线SDK
- 检测频率：每秒5-10帧
- 识别阈值：建议0.85以上
- 检测超时：5秒无有效人脸返回场景榜单

**API接口**

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/user/face/sync | POST | 批量同步用户人脸信息 |
| /api/user/face/add | POST | 添加单个用户人脸 |

| /api/user/face/clear | DELETE | 清空本地人脸库 |

### 3.4 进球检测模块

**功能描述**
- 实时采集光电传感器信号
- 脉冲边沿检测与去抖
- 进球计数与状态同步

**硬件参数**
- 传感器型号：世宇KL-GY40JMFS1
- 输出类型：NPN（遮挡输出低电平）
- 工作电压：10-30V DC
- 信号线：3根线（电源+信号+GND）

**信号特性**
- 无物体遮挡时：输出线悬空（高阻态）
- 有物体遮挡时：输出线导通到GND（低电平，约0V）
- 进球瞬间：球遮挡光束 -> 输出从高阻变为低电平 -> 产生下降沿脉冲

**技术方案**

```
┌─────────────────────────────────────────────────────┐
│              USB Serial 通信协议                      │
├─────────────────────────────────────────────────────┤
│  波特率：    9600                                    │
│  数据位：    8                                       │
│  停止位：    1                                       │
│  校验位：    None                                    │
│  流控：      None                                    │
└─────────────────────────────────────────────────────┘
```

**信号采集流程**
1. USB虚拟串口连接
2. 监听数据流（检测低电平）
3. 下降沿触发中断
4. 软件去抖（50ms，可配置）
5. 进球计数+1
6. 同步到游戏核心模块

**关键参数**
- 串口设备：/dev/ttyUSB0（自动检测）
- 读取间隔：10ms
- 去抖延时：50ms（可配置）
- 超时判定：连续低电平超过1s视为卡球

### 3.5 游戏核心模块

**功能描述**
- 双模式游戏控制
- 实时计分统计
- 游戏状态管理
- 超时自动返回场景榜单

**模式定义**

| 模式 | 参数 | 说明 |
|------|------|------|
| 倒计时模式 | 默认60秒 | 在限定时间内投中最多球 |
| 定数模式 | 默认20球 | 投完固定数量球所用时间最短 |

**完整状态机**

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|----------|----------|
| SCENE_RANK | 场景榜单 | 系统启动 / 超时返回 | 检测到人脸 |
| FACE_DETECT | 人脸检测 | 摄像头检测到人脸 | 识别完成/5秒超时 |
| FACE_RECOGNIZED | 已注册用户 | 比对成功 | 进入待机 |
| FACE_VISITOR | 游客模式 | 比对失败/未注册 | 进入待机 |
| STANDBY | 待机主界面 | 身份确认完成 | 握拳/举手/30秒超时 |
| MODE_SELECT | 模式选择 | 待机握拳 | 模式切换完成 |
| GAME_STARTING | 启动倒计时 | 待机举手 | 321结束 |
| GAME_PLAYING | 游戏进行 | 倒计时结束 | 时间到/球数到 |
| GAME_ENDED | 游戏结束 | 条件达成 | 数据显示完成 |
| LEADERBOARD | 榜单展示 | 游戏结束 | 超时30秒 |

- **排名规则**：用时越短排名越高，相同用时命中率高的排名靠前

- 游客身份允许进行游戏
- 游戏成绩计入本地记录

## 附录C：常见问题FAQ

**Q1：断网情况下游戏能否正常运行？**
A：可以。游戏核心功能离线运行，数据暂存本地，联网后自动补传。

**Q2：人脸识别失败怎么办？**
A：切换到游客模式，允许匿名游戏。提示用户联系管理员登记人脸信息。

**Q3：进球检测漏检怎么办？**
A：检查光电传感器安装位置，调整去抖参数（默认50ms），确保球体遮挡红外光束。

**Q4：USB设备断开怎么办？**
A：暂停游戏，提示用户检查USB连接。恢复连接后自动继续。

**Q5：如何更新人脸库？**
A：后台通过API推送人脸特征值，前端自动同步更新。

## 附录D：GPIO投篮感应器模块开发记录

**开发日期**：2026-03-22

### D.1 功能概述

将原模拟投篮（随机间隔800-1200ms，60%命中率）替换为真实GPIO感应器采集，实现真实的投篮数据检测。

### D.2 实现方案

#### 硬件连接
- **GPIO编号**：81（投篮感应器）
- **采集方式**：JNI直连 /sys/class/gpio/

#### 软件架构
```
GameViewModel.startGameTimer()
    │
    ├── ShootSensorManager.initialize()
    │       └── GpioJniWrapper.requestRootPermission()
    │
    ├── ShootSensorManager.startListening()
    │       └── 20ms轮询 → 边沿检测 → 命中判定
    │
    └── processShotResult(isMade)
```

### D.3 命中判定算法

| 触发类型 | 时间窗口 | 判定结果 |
|----------|----------|----------|
| 单次上升沿 | >500ms无二次触发 | 未命中 |
| 二次上升沿 | <500ms间隔 | 命中 |

**原理**：球进筐后下落会触发两次感应器，单次触发视为碰框弹出

### D.4 配置文件

```kotlin
    const val POLLING_INTERVAL_MS = 20L
    const val DEBOUNCE_MS = 300L

}
```
